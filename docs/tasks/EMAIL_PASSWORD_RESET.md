# Задача: Восстановление пароля через Email (Яндекс SMTP)

**Дата создания:** 2026-03-16
**Дата выполнения:** 2026-03-16

## Цель

Добавить отправку кода восстановления пароля на email пользователя через Яндекс SMTP.
Сейчас коды восстановления отправляются только через Telegram-бота.
Email-канал нужен как альтернатива для пользователей без Telegram.

## Учётные данные

- **Почта:** vcvet.info@yandex.ru
- **SMTP-хост:** smtp.yandex.ru
- **SMTP-порт:** 465 (SSL)
- **Пароль приложения:** _(получить от владельца, сохранить в .env на сервере)_

## Предварительные шаги (владелец аккаунта)

1. Включить двухфакторную аутентификацию в Яндексе (если не включена)
2. Создать "Пароль приложения" для почты: Яндекс ID → Безопасность → Пароли приложений
3. Передать пароль разработчику (НЕ коммитить в репозиторий)

## План реализации

### 1. Конфигурация (.env)
Добавить переменные:
```env
SMTP_HOST=smtp.yandex.ru
SMTP_PORT=465
SMTP_USER=vcvet.info@yandex.ru
SMTP_PASSWORD=<пароль приложения>
SMTP_FROM_NAME=Вцвет Маркет
```

### 2. Сервис отправки email
Создать `apps/api/services/email.py`:
- Async-отправка через `aiosmtplib`
- HTML-шаблон письма с кодом восстановления
- Retry-логика при ошибках SMTP
- Структурное логирование (structlog)

### 3. Интеграция в эндпоинт восстановления пароля
Текущий эндпоинт `POST /auth/forgot-password` отправляет код через Telegram.
Добавить параллельную/альтернативную отправку через email:
- Если у пользователя привязан Telegram → отправлять в Telegram + email
- Если нет Telegram → отправлять только email

### 4. Зависимости
```
pip install aiosmtplib
```

### 5. Деплой
- Добавить SMTP-переменные в `.env` на сервере
- Обновить `.env.example` в репозитории (без реальных паролей)

Небольшая задача: ~2-3 файла, без миграций БД.
